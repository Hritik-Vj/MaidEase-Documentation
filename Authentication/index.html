    Authentication & Authorization Overview¶

    Authentication¶

    MaidEase uses a JWT-based authentication system with both Access Tokens and Refresh Tokens.
    This ensures secure session handling between the React frontend and the FastAPI backend.

    When a user logs in, the backend generates:

    • Access Token (short lifespan)
    • Refresh Token (long lifespan)

    Both tokens are returned to the frontend and stored in localStorage.

    All protected API calls must include the Access Token in:

    Authorization: Bearer <access_token>

    Authentication Flow¶

    1. User Registration¶

      Users register with:
    2. email
    3. full_name
    4. phone_number
    5. role (customer or maid)
    6. password

    Backend stores passwords using Argon2 hashing.

    1. Login¶

    2. User submits email + password
    3. Backend verifies credentials
    4. Backend returns:
      • access_token
      • refresh_token
      • token_type: bearer

    5. Frontend stores tokens using setTokens().

    6. Accessing Protected Routes¶

    7. Axios interceptor attaches the Access Token automatically
    8. Backend validates token using python-jose

    9. User identity is extracted from token payload (user_id, role)

    10. Automatic Token Refresh¶

      When Access Token expires:

    11. Axios sends request to /auth/refresh
    12. Backend verifies refresh token
    13. New Access Token is issued

    14. Frontend updates localStorage

    15. Logout¶

    16. Frontend clears both tokens
    17. User session ends

    Authorization (RBAC — Role-Based Access Control)¶

    MaidEase uses role-based authorization.
    Users have one of two roles defined in the database:

    • customer
    • maid
